## ğŸ’» Technical Breakdown

### PHP Backend (index.php at the top)

The PHP section executes only when an HTTP POST request is received.

Input Sanitization: Uses trim(), stripslashes(), and htmlspecialchars() to sanitize user inputs before processing.

Validation: Checks for required fields, valid email format (FILTER_VALIDATE_EMAIL), and minimum phone number length.

Error Handling: If validation fails, it sets the HTTP response code to 400 (Bad Request) and returns a styled error card listing all issues.

Success Handling: If validation passes, it sets the HTTP response code to 200 (OK) and returns a styled success card with the submitted data summary.

### JavaScript Frontend (<script> tag at the bottom)

The JavaScript section handles the user interaction and the asynchronous communication.

##### Event Listener: Attaches a listener to the form's submit event, calling e.preventDefault() to stop default browser submission.

##### Form Serialization: Converts the form data into a URLSearchParams object for the fetch request body.

##### AJAX fetch: Submits the data to the same index.php file using the POST method.

##### Response Handling:

If response.ok is true (status 200), it displays the success message and hides the original form.

If the status is 400, it displays the PHP-generated error message.

Includes a basic exponential backoff retry mechanism for better network reliability.

##### UI Updates: Manages the loading spinner and disables/re-enables the submit button during the request lifecycle.
